1. 웹화면 디자인 정하기 ( 가장 어려움 하..................)

절대 절대 양산형 챗봇 디자인은 금지 xxxxxxxxx 가장 중요 


- 화면 구성
  - RPA 관련 화면과 chatbot 화면, OCR 화면 등 관리 배치도 고민할 것
  - 관리자 페이지 중점 디자인 (MCP 연동 염두)

  - 회원가입은 메일 연동 (로그인도)

  - 디자인에 API 연동을 염두해둘 것
  - GPT 같은 API 모델도 사용 가능하게 하면 좋을듯

- 기술 스택
  - 백엔드: FastAPI로 백단 연결해서 사용함
  - 배포: 벌셀로 진행
  - 프론트 관련 DB: MongoDB 사용
  - 벡터 DB: ChromaDB 사용
  - 프로세스 관리: FastAPI를 PM2로 관리하여 자동실행이 될 수 있게 사용


2. 기능구현 정리하기

RAG LLM -Starter 
- 임베딩 할때 권한별 조회 접근 다르게 할 것
- 출력시 이미지 나오고 하단에 출처 링크로 원본문서 사용 가능하게 할 것
- 임베딩시 이미지가 있다면 자동 OCR 처리 기반 임베딩 (API 사용 X, 직접 구현)
- api 연동기능 Starter




파인튜닝 -Enterprise
- 해당 특정 알고리즘 파인튜닝 진행

Agent
- RPA 연동으로 세금계산서 자동 발행 
- ERP 연동 수정, 작성, 삭제
- 생각나거나 추가 진행
- rpa 가 들어가는 agent는 Enterprise 로 
- rpa가 빠진 agent 는 Pro 


보고서 만들어주는거  회의내용 요약해주고  (아키텍쳐 그려주고 캔버스 작업과 연동까지 ) 



OCR
- 임베딩을 위한 범용성 진행
- 특정 기능을 위한 OCR 진행
- 해당 mcp로  챗봇연동으로 파서 진행 챗봇 자체에 ocr 연동
- 그냥 단순 임배딩은 Starter
- ocr 문서 전처리 는 pro 
- 업체 전용 ocr 처리 Enterprise



STT,tts -Enterprise 
- 보이스 연동

MCP
